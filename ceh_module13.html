üñ•Ô∏è Web Server Fundamentals

üåê What is a Web Server?

A web server is a software application that serves web content to clients over HTTP/HTTPS. Think of it as a digital librarian that receives requests for specific books (web pages) and delivers them to visitors.

Key Functions:

  • Process HTTP/HTTPS requests from clients
  • Serve static content (HTML, CSS, images)
  • Execute dynamic content (PHP, ASP.NET, JSP)
  • Handle authentication and authorization
  • Manage SSL/TLS encryption
  • Log activities and monitor performance

üî• HTTP/2 Continuation Flood Attack

A sophisticated new attack technique targeting HTTP/2 servers:

‚ö° Attack Mechanism

  1. Send initial HTTP/2 HEADERS frame
  2. Follow with multiple CONTINUATION frames
  3. Exhaust server memory with incomplete headers
  4. Cause service degradation or crash

üéØ Vulnerable Servers

  • NGINX (unpatched installations)
  • Microsoft IIS with HTTP/2 enabled
  • Various reverse proxy configurations

üåä Slowloris Attack Technique

A low-bandwidth, high-impact DoS attack method:

üìã Attack Process

  1. Connection Establishment: Open multiple HTTP connections
  2. Partial Headers: Send incomplete HTTP headers slowly
  3. Keep-Alive: Periodically send additional headers
  4. Resource Exhaustion: Exhaust server's connection pool
  5. Service Denial: Prevent legitimate users from connecting
