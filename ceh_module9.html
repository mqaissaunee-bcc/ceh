🔧 DoS Attack Fundamentals

📖 What is a Denial of Service Attack?

A Denial of Service (DoS) attack is a malicious attempt to disrupt the normal traffic of a targeted server, service, or network by overwhelming the target or its surrounding infrastructure with a flood of Internet traffic.

🔍 Key Characteristics:

  • Intent: Make services unavailable to legitimate users
  • Method: Resource exhaustion or service disruption
  • Scope: Can target servers, networks, applications, or services
  • Duration: Can range from minutes to days

⚔️ DoS vs DDoS: Understanding the Difference

🎯 DoS Attack

  • Single attack source
  • Limited bandwidth/resources
  • Easier to trace and block
  • Lower impact potential
  • Simpler to execute
  • Example: Single computer flooding target

💥 DDoS Attack

  • Multiple attack sources (botnet)
  • Massive combined bandwidth
  • Difficult to trace all sources
  • High impact potential
  • Complex coordination required
  • Example: 100,000 infected machines

⚔️ DoS Attack Types & Techniques

🌐 Network Layer Attacks

Attacks targeting the network infrastructure and protocols (Layers 3-4 of OSI model).

🔥 SYN Flood Attack

Exploits TCP three-way handshake by sending numerous SYN requests without completing connections.

  • Exhausts connection table
  • Half-open connections
  • Memory exhaustion

📡 ICMP Flood (Ping Flood)

Overwhelms target with ICMP Echo Request packets, consuming bandwidth and processing power.

  • Bandwidth saturation
  • Processing overhead
  • Network congestion

⚡ UDP Flood Attack

Sends large volumes of UDP packets to random ports, forcing target to check for listening applications.

  • Port unreachable responses
  • Bandwidth consumption
  • Resource exhaustion

🧩 IP Fragmentation Attacks

Exploits IP packet fragmentation to overwhelm reassembly buffers and processing.

  • Teardrop attacks
  • Fragment overlap
  • Reassembly exhaustion

🖥️ Application Layer Attacks

Sophisticated attacks targeting specific applications and services (Layer 7 of OSI model).

🌐 HTTP Flood Attack

Sends legitimate-looking HTTP requests to overwhelm web servers and applications.

  • GET/POST request floods
  • Session exhaustion
  • Database overload

🐌 Slowloris Attack

Keeps connections open by sending partial HTTP requests, exhausting connection pool.

  • Low bandwidth usage
  • Connection pool exhaustion
  • Stealthy approach

📡 DNS Amplification

Uses DNS servers to amplify attack traffic by spoofing victim's IP address.

  • Traffic amplification (up to 70x)
  • IP spoofing
  • Recursive DNS abuse

🎯 Application-Specific

Targets specific application vulnerabilities or resource-intensive operations.

  • Database query floods
  • Search function abuse
  • File upload floods

💥 Distributed Denial of Service (DDoS)

🤖 Botnet Architecture

Understanding the infrastructure behind large-scale DDoS attacks.

1. 🦠 Infection Phase

Malware spreads through various vectors (email, downloads, exploits) to create infected machines (bots).

  • Email attachments with malware
  • Drive-by downloads from compromised websites
  • Exploit kits targeting vulnerabilities
  • Social engineering tactics

2. 📡 Command & Control (C&C)

Infected machines connect to command and control servers for instructions and coordination.

  • IRC (Internet Relay Chat) channels
  • HTTP/HTTPS communication
  • P2P (Peer-to-Peer) networks
  • Social media platforms (rare)

3. 🎯 Coordination Phase

Botmaster coordinates the attack, sending target information and attack instructions to the botnet.

  • Target IP addresses and domains
  • Attack type and parameters
  • Timing and duration
  • Traffic shaping instructions

4. 💥 Attack Execution

Coordinated attack begins with potentially millions of bots targeting the victim simultaneously.

  • Simultaneous traffic generation
  • Multiple attack vectors
  • Geographic distribution
  • Traffic volume amplification

📈 DDoS Attack Categories

📊 Volumetric Attacks

Overwhelm bandwidth capacity of target or intermediate network infrastructure.

  • UDP Floods: 65.1% of attacks
  • ICMP Floods: 18.2% of attacks
  • Spoofed Packet Floods: 16.7%
  • Peak Size: Up to 3.8 Tbps

🔗 Protocol Attacks

Exploit weaknesses in server resources or intermediate communication equipment.

  • SYN Floods: 43.2% of protocol attacks
  • Ping of Death: 22.1% of attacks
  • Smurf Attacks: 19.8% of attacks
  • Impact: Server/firewall exhaustion

🌐 Application Layer Attacks

Target specific applications and services, often appearing as legitimate traffic.

  • HTTP Floods: 56.7% of app attacks
  • DNS Query Floods: 23.4%
  • SSL/TLS Attacks: 12.9%
  • Detection: Most difficult to identify

🎯 Multi-Vector Attacks

Combine multiple attack methods simultaneously for maximum impact and defense evasion.

  • Combined Attacks: 38.9% of incidents
  • Layer 3/4 + Layer 7: Most common
  • Success Rate: 73% higher
  • Defense: Requires multi-layered protection

🛠️ DoS/DDoS Tools & Techniques

⚡ Low-Level Network Tools

Command-line tools for generating various types of network traffic for testing purposes.

🔧 hping3

Purpose: TCP/IP packet assembler/analyzer

  • Custom packet crafting
  • SYN flood testing
  • Port scanning capabilities
  • Traceroute-like functionality
hping3 -S -p 80 --flood target.com

🐍 Scapy

Purpose: Python packet manipulation library

  • Custom protocol implementation
  • Packet crafting and analysis
  • Advanced attack scenarios
  • Scriptable automation
send(IP(dst="target")/TCP(dport=80))

🗺️ Nmap

Purpose: Network discovery and security auditing

  • TCP/UDP flood scripts
  • Service enumeration
  • Vulnerability detection
  • Network mapping
nmap --script dos -Pn target.com

🌐 Application Layer Tools

Specialized tools for testing web applications and HTTP services.

💥 LOIC (Low Orbit Ion Cannon)

Purpose: Simple DOS testing tool

  • HTTP/TCP/UDP floods
  • GUI-based interface
  • Voluntary botnet participation
  • Basic stress testing

⚠️ For authorized testing only

🚀 HOIC (High Orbit Ion Cannon)

Purpose: Advanced HTTP flooding tool

  • Multi-threaded HTTP requests
  • Scriptable "booster" modules
  • Multiple target support
  • Advanced evasion techniques

⚠️ Requires proper authorization

🐌 Slowloris

Purpose: Low-bandwidth connection exhaustion

  • Partial HTTP header attacks
  • Connection pool exhaustion
  • Minimal bandwidth usage
  • Stealthy attack profile
perl slowloris.pl -dns target.com

🏰 Siege

Purpose: HTTP load testing utility

  • Concurrent user simulation
  • Performance benchmarking
  • Configuration file support
  • Detailed reporting
siege -c 100 -t 60s http://target.com

🤖 Advanced DDoS Platforms

Sophisticated platforms used by researchers and security professionals for comprehensive testing.

🎯 Metasploit Framework

⚡ DDoSIM

🔐 THC-SSL-DOS

Purpose: SSL/TLS handshake exhaustion

  • SSL handshake flooding

  • HTTPS service targeting
thc-ssl-dos target.com 443

📊 Traffic Generation Techniques

🔥 Direct Attacks

  • Single source to target
  • High bandwidth requirements
  • Easy to trace and block
  • Limited scalability
  • Example: Ping flood from one machine

🌐 Reflection Attacks

  • Uses third-party servers
  • IP address spoofing required
  • Traffic amplification possible
  • Harder to trace back
  • Example: DNS reflection attacks

⚡ Amplification Attacks

  • Small request, large response
  • Amplification factors up to 4,670x
  • Minimal attacker bandwidth
  • Maximum victim impact
  • Example: NTP, DNS, LDAP amplification

🛡️ DoS/DDoS Countermeasures

🏗️ Infrastructure-Level Protection

Fundamental network and infrastructure hardening measures.

📈 Bandwidth Over-provisioning

Maintain excess bandwidth capacity to absorb attack traffic.

  • 10x normal capacity recommendation
  • Multiple ISP connections
  • Geographic distribution
  • Automatic scaling capabilities

🚫 Traffic Filtering

Block malicious traffic at network perimeter.

  • IP blacklisting
  • Rate limiting
  • Ingress/egress filtering
  • Protocol-specific filters

⚖️ Load Balancing

Distribute traffic across multiple servers and data centers.

  • Geographic load balancing
  • Anycast routing
  • Health monitoring
  • Automatic failover

🔄 System Redundancy

Multiple backup systems and failover mechanisms.

  • Hot/cold standby servers
  • Database replication
  • Network path diversity
  • Disaster recovery sites

🛠️ Technical Countermeasures

🍪 SYN Cookies

Purpose: Mitigate SYN flood attacks

  • Eliminates connection state storage
  • Cryptographic sequence numbers
  • No memory allocation until ACK
  • Transparent to legitimate clients
echo 1 > /proc/sys/net/ipv4/tcp_syncookies

⏱️ Rate Limiting

Purpose: Control request frequency per source

  • Per-IP connection limits
  • Request frequency throttling
  • Bandwidth allocation control
  • Adaptive threshold adjustment
iptables -A INPUT -p tcp --dport 80 -m limit --limit 10/min -j ACCEPT

🕳️ Blackhole Routing

Purpose: Route malicious traffic to null interface

  • BGP blackhole communities
  • Remote triggered blackhole (RTBH)
  • Automatic threat intelligence
  • ISP-level coordination
route add 192.168.1.0/24 blackhole

🤖 CAPTCHA Systems

Purpose: Differentiate humans from bots

  • Image-based challenges
  • Audio alternatives
  • reCAPTCHA v3 risk scoring
  • Behavioral analysis
if (suspiciousActivity) { showCaptcha(); }

📋 Incident Response Framework

1. 🚨 Detection & Analysis

Identify attack patterns and assess impact severity.

  • Monitor network traffic baselines
  • Analyze traffic patterns and anomalies
  • Classify attack type and vectors
  • Assess business impact and severity

2. 🛡️ Containment & Mitigation

Implement immediate countermeasures to reduce attack impact.

  • Activate DDoS protection services
  • Implement traffic filtering rules
  • Scale infrastructure resources
  • Contact ISP for upstream filtering

3. 📞 Communication

Coordinate internal and external communication efforts.

  • Notify incident response team
  • Update stakeholders and management
  • Communicate with customers/users
  • Coordinate with law enforcement if needed

4. 🔍 Recovery & Analysis

Restore normal operations and conduct post-incident analysis.

  • Gradually restore affected services
  • Monitor for attack resumption
  • Document lessons learned
  • Update incident response procedures
