Wireless Network Fundamentals

Radio Frequency Fundamentals

Wireless networks operate using electromagnetic waves in the radio frequency spectrum. Understanding RF principles is essential for both attacking and defending wireless networks.

Frequency Bands

  • 2.4 GHz: 802.11b/g/n, longer range, more interference
  • 5 GHz: 802.11a/n/ac/ax, shorter range, less crowded
  • 6 GHz: Wi-Fi 6E, newest band with more channels
  • 60 GHz: 802.11ad/ay, very short range, high speed

Antenna Types

  • Omnidirectional: 360-degree coverage
  • Directional: Focused beam, increased range
  • Yagi: High-gain directional antenna
  • Parabolic: Very high gain, long-distance

Network Topologies

Infrastructure Mode

Centralized topology with access points managing client connections. Most common in enterprise environments.

  • Central access point coordination
  • SSID broadcasting
  • Authentication and encryption
  • Network traffic routing

Ad-Hoc Mode

Peer-to-peer network without central access point. Devices communicate directly with each other.

  • Direct device communication
  • No central authority
  • Limited security options
  • Temporary network setups

Wireless Security Protocols

Evolution of Wireless Security

Wireless security protocols have evolved to address vulnerabilities and provide stronger protection. Understanding each protocol's strengths and weaknesses is crucial for effective security assessment.

WEP (Wired Equivalent Privacy)

Status: Deprecated (2004)

  • Encryption: RC4 with 40/104-bit keys
  • Key Management: Static keys
  • Vulnerabilities: IV attacks, weak implementation
  • Crack Time: Minutes to hours
  • Security Rating: ★ (Broken)

WPA (Wi-Fi Protected Access)

Status: Legacy (2003)

  • Encryption: TKIP with RC4
  • Key Management: Dynamic key rotation
  • Vulnerabilities: TKIP attacks, WPS flaws
  • Crack Time: Hours to days
  • Security Rating: ★★ (Weak)

WPA2 (802.11i)

Status: Current Standard (2004)

  • Encryption: AES-CCMP
  • Key Management: Robust key derivation
  • Vulnerabilities: KRACK, dictionary attacks
  • Crack Time: Days to years
  • Security Rating: ★★★★ (Strong)

WPA3 (Latest)

Status: Next Generation (2018)

  • Encryption: AES-256, SAE
  • Key Management: Perfect Forward Secrecy
  • Vulnerabilities: Minimal known issues
  • Crack Time: Currently infeasible
  • Security Rating: ★★★★★ (Excellent)

WPA3 Security Enhancements

  • SAE (Simultaneous Authentication of Equals): Replaces PSK with stronger handshake
  • Perfect Forward Secrecy: Past sessions remain secure even if key is compromised
  • Enhanced Protection: Stronger brute-force attack resistance
  • Individualized Encryption: Unique encryption for each client in public networks

Authentication Methods

Personal (PSK)

  • Pre-shared key authentication
  • Single passphrase for all users
  • Suitable for home/small office
  • Vulnerable to dictionary attacks

Enterprise (802.1X/EAP)

  • RADIUS server authentication
  • Individual user credentials
  • Certificate-based authentication
  • Enhanced security and auditing

Wireless Attack Techniques

Attack Classifications

Wireless attacks can be categorized into several types based on their objectives and methodologies. Understanding these categories helps in developing comprehensive defense strategies.

Passive Attacks

Eavesdropping and monitoring wireless traffic without actively interfering with network operations.

  • Wireless sniffing and packet capture
  • Traffic analysis and pattern recognition
  • Signal strength monitoring
  • Network reconnaissance

Active Attacks

Direct interference with wireless networks through injection, jamming, or manipulation of communications.

  • Packet injection and replay attacks
  • Deauthentication and disassociation
  • Evil twin access point setup
  • Man-in-the-middle attacks

Denial of Service

Disrupting wireless network availability through various interference and flooding techniques.

  • RF jamming and interference
  • Authentication flood attacks
  • Association flood attacks
  • Power exhaustion attacks

Encryption Attacks

Breaking or bypassing wireless encryption mechanisms to access protected communications.

  • WEP key recovery attacks
  • WPA/WPA2 handshake capture
  • Dictionary and brute-force attacks
  • Rainbow table attacks

WEP Attack Techniques

WEP: The Broken Protocol

WEP (Wired Equivalent Privacy) was officially deprecated in 2004 due to fundamental cryptographic flaws. However, legacy systems still use WEP, making these attacks relevant for security assessments.

WEP Vulnerabilities

IV (Initialization Vector) Flaws

  • 24-bit IV creates predictable patterns
  • IV reuse after ~5,000 packets
  • Weak IV sequences reveal key bits
  • Statistical analysis possible

Key Management Issues

  • Static key sharing among all users
  • No key rotation mechanism
  • Weak key derivation function
  • Poor entropy in key generation

WEP Attack Methods

FMS Attack (Fluhrer-Mantin-Shamir)

Method: Statistical analysis of weak IVs

Requirements: 500,000+ unique IVs

Success Rate: High with sufficient data

Time Required: 2-8 hours passive collection

KoreK Attack

Method: Enhanced statistical approach

Requirements: 300,000+ unique IVs

Success Rate: Higher than FMS

Time Required: 1-4 hours passive collection

PTW Attack (Pyshkin-Tews-Weinmann)

Method: Optimized statistical analysis

Requirements: 40,000+ ARP packets

Success Rate: ~50% with minimal data

Time Required: 30 minutes - 2 hours

Chop-Chop Attack

Method: Interactive packet modification

Requirements: Single encrypted packet

Success Rate: High with cooperation

Time Required: Minutes to decrypt single packet

WEP Cracking Process

2
Fake Authentication

3

  • Packet Injection: Successful fake authentication to AP
  • ARP Replay: Continuous ARP request/response pairs
  • Key Recovery: 40-bit or 104-bit WEP key revealed

WPA/WPA2 Attack Techniques

WPA/WPA2 Security Overview

WPA and WPA2 addressed many WEP vulnerabilities but introduced new attack vectors. Understanding these attacks is crucial for modern wireless security assessment.

4-Way Handshake Process

  1. ANonce: AP sends random number
  2. SNonce + MIC: Client responds with its nonce
  3. GTK + MIC: AP sends group temporal key
  4. ACK: Client acknowledges completion

Vulnerability: Handshake can be captured and attacked offline

KRACK Attack (2017)

CVE-2017-13077 to 13088

  • Key reinstallation attack
  • Exploits 4-way handshake
  • Affects all WPA2 implementations
  • Allows packet decryption and injection

WPA/WPA2 Attack Methods

Handshake Capture Attack

Objective: Capture 4-way authentication handshake

Method: Monitor association or force deauth

Requirements: Client must (re)connect during capture

Success Rate: 95%+ with patience

Dictionary Attack

Objective: Crack PSK using wordlist

Method: Offline brute-force against captured handshake

Requirements: Good wordlist and computing power

Success Rate: Depends on password complexity

WPS Attack

Objective: Exploit Wi-Fi Protected Setup flaws

Method: PIN brute-force or Pixie Dust attack

Requirements: WPS enabled on target AP

Success Rate: 90%+ on vulnerable devices

PMKID Attack

Objective: Extract PMKID without handshake

Method: Request PMKID from AP without association

Requirements: AP that responds with PMKID

Success Rate: 70%+ on modern routers

Modern Defense Mechanisms

  • WPS Lockout: Many modern routers disable WPS after failed attempts
  • Rate Limiting: Deauthentication attack protection
  • Strong PSKs: Long, complex passwords resist dictionary attacks
  • 802.11w (PMF): Management frame protection prevents deauth attacks
  • WPA3: SAE protocol eliminates offline dictionary attacks

Wireless Security Countermeasures

Defense-in-Depth Strategy

Effective wireless security requires multiple layers of protection. A comprehensive approach addresses technical, procedural, and physical security aspects.

Encryption Controls

  • WPA3 Implementation: Deploy latest security standard
  • Strong PSKs: 15+ character complex passwords
  • Enterprise Authentication: 802.1X/EAP with certificates
  • Regular Key Rotation: Periodic password changes

Access Point Security

  • Disable WPS: Prevent PIN brute-force attacks
  • Hide SSID: Reduce casual discovery attempts
  • MAC Filtering: Whitelist authorized devices
  • Firmware Updates: Regular security patches

Network Segmentation

  • Guest Networks: Isolate visitor traffic
  • VLAN Separation: Segregate wireless from wired
  • Firewall Rules: Restrict inter-network communication
  • DMZ Placement: Contain wireless networks

Monitoring & Detection

  • WIDS/WIPS: Intrusion detection and prevention
  • Rogue AP Detection: Unauthorized access point identification
  • Anomaly Detection: Unusual traffic pattern analysis
  • Log Analysis: Security event correlation

Specific Attack Mitigations

WEP Attack Prevention

  • Complete Elimination: Replace all WEP-capable devices
  • Legacy System Isolation: Air-gap critical WEP systems
  • VPN Tunneling: Encrypt traffic over WEP networks
  • Regular Audits: Scan for WEP usage

Handshake Attack Mitigation

  • 802.11w (PMF): Management frame protection
  • Strong PSKs: Resist dictionary attacks
  • Deauth Protection: Rate limiting and blocking
  • Client Education: Avoid unknown networks

WPS Attack Prevention

  • Disable WPS: Turn off entirely if not needed
  • WPS Lockout: Enable automatic lockout features
  • PIN Randomization: Regular PIN changes
  • Alternative Setup: Manual configuration preferred

Physical Security

  • AP Placement: Secure mounting locations
  • Signal Containment: RF shielding where needed
  • Power Protection: UPS and surge protection
  • Tamper Detection: Physical access monitoring

Wireless Security Knowledge Test

Test your understanding of wireless network security concepts, attack techniques, and countermeasures.

1. Which wireless security protocol is most vulnerable to IV attacks?
A) WPA3
B) WPA2
C) WEP
D) WPA
Explanation: WEP is highly vulnerable to IV (Initialization Vector) attacks due to its weak implementation of RC4 encryption and reuse of IVs after approximately 5,000 packets.
2. What is the primary advantage of the PMKID attack over traditional handshake capture?
A) Faster cracking speed
B) No client interaction required
C) Works against WPA3
D) Bypasses MAC filtering
Explanation: PMKID attacks can extract the PMKID directly from access points without requiring client authentication or deauthentication attacks.
3. Which attack technique exploits the 4-way handshake in WPA2?
A) KRACK (Key Reinstallation Attack)
B) Caffe Latte Attack
C) Fragmentation Attack
D) Chop-Chop Attack
Explanation: KRACK exploits vulnerabilities in the 4-way handshake by forcing key reinstallation, allowing attackers to decrypt and inject packets.
4. What is the minimum number of unique IVs typically required for a successful PTW attack on WEP?
A) 10,000
B) 40,000
C) 100,000
D) 500,000
Explanation: The PTW attack requires approximately 40,000 ARP packets (unique IVs) for effective WEP key recovery, making it much more efficient than older attacks.
5. Which protocol provides Perfect Forward Secrecy in wireless networks?
A) WEP
B) WPA
C) WPA2
D) WPA3
Explanation: WPA3 implements Perfect Forward Secrecy through SAE (Simultaneous Authentication of Equals), ensuring past sessions remain secure even if current keys are compromised.
6. What does the term "Evil Twin" refer to in wireless security?
A) Duplicate MAC addresses
B) Rogue access point mimicking legitimate network
C) Cloned wireless cards
D) Identical SSID names
Explanation: An Evil Twin is a rogue access point that mimics a legitimate wireless network to trick users into connecting and potentially stealing credentials.
7. Which tool is specifically designed for WPS PIN brute-force attacks?
A) Airodump-ng
B) Reaver
C) Wireshark
D) Kismet
Explanation: Reaver is specifically designed to perform WPS PIN brute-force attacks and includes implementations of various WPS vulnerabilities including Pixie Dust attacks.
8. What is 802.11w designed to protect against?
A) Data frame attacks
B) Management frame attacks
C) Control frame attacks
D) Beacon frame attacks
Explanation: 802.11w (Protected Management Frames) protects against management frame attacks such as deauthentication and disassociation attacks by adding integrity protection.
9. Which frequency band is most commonly used for 802.11n networks?
A) 2.4 GHz and 5 GHz
B) 2.4 GHz only
C) 5 GHz only
D) 6 GHz
Explanation: 802.11n operates in both 2.4 GHz and 5 GHz frequency bands, providing flexibility in deployment and helping avoid interference.
10. What is the primary weakness exploited in WPS Pixie Dust attacks?
A) Weak random number generation
B) Poor implementation of nonce generation
C) Insufficient PIN complexity
D) Lack of rate limiting
Explanation: Pixie Dust attacks exploit poor implementation of nonce generation in WPS, allowing attackers to recover the WPS PIN without brute-forcing all possibilities.
