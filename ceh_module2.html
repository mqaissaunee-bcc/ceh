üéØ Types of Reconnaissance

Understanding Reconnaissance Methodologies

Reconnaissance is the process of gathering information about a target system or organization. It's the foundation of any successful penetration test and can be categorized into two main approaches based on the level of interaction with the target.

ü§´ Passive Reconnaissance

(Indirect Information Gathering)

Approach: No direct interaction with target

Detection Risk: Virtually zero

Legal Status: Generally legal


Techniques:

  • Open Source Intelligence (OSINT)
  • Search engine queries
  • Social media analysis
  • Public records research
  • DNS lookup using third-party tools

Advantage: Undetectable and safe

Disadvantage: Limited information depth

‚ö° Active Reconnaissance

(Direct Target Interaction)

Approach: Direct interaction with target systems

Detection Risk: High - leaves traces

Legal Status: Requires authorization


Techniques:

  • Network scanning (nmap, masscan)
  • Port enumeration
  • Service fingerprinting
  • Direct DNS queries to target
  • Social engineering calls/emails

Advantage: Detailed, real-time information

Disadvantage: Detectable and potentially illegal

üîÑ The Reconnaissance Cycle

1. Information Gathering ‚Üí 2. Analysis ‚Üí 3. Correlation ‚Üí 4. Validation ‚Üí 5. Reporting

Effective reconnaissance follows a systematic cycle where each phase builds upon the previous one, creating a comprehensive intelligence picture of the target.

‚öñÔ∏è Legal and Ethical Considerations

‚ö†Ô∏è WARNING: Always ensure you have proper authorization before conducting active reconnaissance. Passive reconnaissance using publicly available information is generally legal, but active reconnaissance can constitute unauthorized access.

Best Practices:

  • Start Passive: Exhaust all passive techniques before moving to active
  • Document Everything: Keep detailed records of sources and methods
  • Respect Privacy: Avoid gathering personal information unnecessarily
  • Stay Current: Laws and regulations change - stay informed

üöÄ Advanced Google Hacking Techniques

Google hacking (also known as Google dorking) uses advanced search operators to find specific information that may not be easily accessible through normal searches. These techniques can reveal sensitive information, system vulnerabilities, and detailed organizational intelligence.

intitle: Operator

Example: intitle:"index of" password

Purpose: Find pages with specific titles

Risk: Can reveal directory listings with sensitive files

filetype: Operator

Example: filetype:pdf confidential

Purpose: Search for specific file types

Risk: May expose confidential documents

site: Operator

Example: site:company.com login

Purpose: Limit search to specific domain

Risk: Can reveal hidden login pages

inurl: Operator

Example: inurl:admin

Purpose: Find URLs containing specific terms

Risk: May expose administrative interfaces

cache: Operator

Example: cache:example.com

Purpose: View Google's cached version

Risk: Can reveal deleted or modified content

ext: Operator

Example: ext:sql + ext:txt

Purpose: Search for file extensions

Risk: May find database dumps or config files

üåê DNS & WHOIS Footprinting

üîç DNS Intelligence Gathering

DNS (Domain Name System) footprinting reveals critical infrastructure information about target organizations. DNS records contain a wealth of information including IP addresses, mail servers, subdomains, and network topology.

1. WHOIS Lookup

Gather domain registration information, contacts, and nameservers

2. DNS Record Enumeration

Collect A, AAAA, MX, NS, TXT, and other DNS records

3. Subdomain Discovery

Find hidden subdomains using brute force and zone transfers

4. Reverse DNS Lookup

Map IP addresses back to hostnames and domains

5. Zone Transfer Attempts

Try to extract complete zone files from DNS servers

üìã Key DNS Record Types

A Record

Maps domain to IPv4 address

Example: example.com ‚Üí 192.168.1.1

AAAA Record

Maps domain to IPv6 address

Example: example.com ‚Üí 2001:db8::1

MX Record

Mail exchange servers

Example: mail.example.com

NS Record

Authoritative nameservers

Example: ns1.example.com

TXT Record

Text information, SPF, DKIM

Example: "v=spf1 include:_spf.google.com ~all"

CNAME Record

Canonical name alias

Example: www.example.com ‚Üí example.com
